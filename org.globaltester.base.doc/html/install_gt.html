Installing GlobalTester Plug-ins

GlobalTester Plug-ins are installed by using the standard Eclipse Install Software mechanism.

From the Eclipse main menu select Help, Install New Software...

 

The dialog Available Software will appear. Just click the Add... button to add a new Update Site.

 

You will then be prompted to select the location which should be searched for new software. Please select the plug-in file you downloaded to your harddrive.

If you have not yet downloaded your plug-in you can download a ZIP-file containing a copy from our download site at http://www.globaltester.org or (depending on your subscription) - remember to Login or the downloads are hidden. This ZIP-file includes all files that are present on the update site together with the needed meta files to create a local update site.

After making these entries click OK.

 

 

 

Note

You may also need to install "Unlimited Strength Jurisdiction Policy Files". You can download them directly from Oracle. Please make sure to download a version that fits with your installed Java Runtime Environment. Just copy the files into \Java\jreX\lib\security folder and overwrite the existing files.

 

 

Now you see your new update site available and you can select which features to install from the side. In the screenshot below you see an Eclipse installation an example update site. The Features you can select depend on the chosen update site. Some of them may not be accessible for you depending on your subscription. Select the features you want to use. Now press the Next button.

 

The following dialog shows you which packages need to be installed for installing. Press Next to accept this.

 

You will then be prompted to agree to the license conditions for GlobalTester. The installation is started by clicking on Finish.

 

After installing GlobalTester Plug-ins Eclipse must be restarted. Select Yes when you see the following screen:

 

After the restart you should check your installation.
