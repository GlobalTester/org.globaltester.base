After you created a new plugin to implement new functionality within GlobalTester you most certainly want to implement a test fragment also. As detailed in our development guidelines a test fragment is required for each plugin that exists in GlobalTester repository.

Add launch configurations to buckminster scripts

Currently the scripts executed during the CI-Builds on Jenkins are still maintained manually. They are located in com.hjp.globaltester.releng and you need to add lines like the following to them (somewhere after the build command):

junit -l org.globaltester.core.test/org.globaltester.core.test.full.launch --flatXML -o junit/org.globaltester.core.xml

Sometime in the future we plan to replace those scripts by automatically generated ones so that this step will become needless.
